# Voice Notes

Convert audio recordings to structured meeting notes using OpenAI Whisper.

## Features

- **Local Transcription**: Uses OpenAI Whisper running locally (no API costs)
- **Smart Extraction**: Automatically identifies key points, action items, and attendees
- **Multiple Models**: Choose from tiny/base/small/medium/large for speed vs accuracy trade-off
- **Audio Format Support**: Works with .wav, .mp3, .m4a, .flac, .ogg, .webm
- **Voice Summary**: Optionally speaks a summary using voice-mcp TTS
- **Timestamps**: Include timestamps in the transcript

## Installation

### Prerequisites

1. **Python 3.8+**
2. **FFmpeg** (required by Whisper for audio processing)

```bash
# Ubuntu/Debian
sudo apt install ffmpeg

# macOS
brew install ffmpeg

# Windows
winget install ffmpeg
# OR download from https://ffmpeg.org/download.html
```

### Install voice-notes

```bash

cd /mnt/d/_CLAUDE-TOOLS/voice-notes

# Install in development mode
pip install -e .

# OR install dependencies only
pip install -r requirements.txt
```

## Usage

### Basic Usage

```bash
# Transcribe an audio file and generate notes
voice-notes meeting.wav

# Output will be saved to meeting_notes.md
```

### Command Line Options

```bash
voice-notes [OPTIONS] AUDIO_FILE

Options:
  -o, --output PATH       Output markdown file (default: <input>_notes.md)
  -m, --model MODEL       Whisper model: tiny, base, small, medium, large
                          (default: base)
  -l, --language CODE     Force language (e.g., 'en', 'es'). Auto-detected if
                          not specified.
  --speak                 Speak a summary after processing
  --no-transcript         Don't include full transcript in output
  --timestamps            Include timestamps in the transcript
  -v, --verbose           Show detailed transcription progress
  --version               Show version and exit
  --help                  Show help message
```

### Examples

```bash
# Use a more accurate model (slower)
voice-notes meeting.mp3 --model medium

# Save to specific file
voice-notes recording.wav -o project-notes.md

# Get voice summary when done
voice-notes standup.wav --speak

# Force English and show progress
voice-notes podcast.mp3 --language en --verbose

# Just the extracted notes, no full transcript
voice-notes call.wav --no-transcript

# Include timestamps in output
voice-notes interview.wav --timestamps
```

## Model Selection Guide

| Model  | Speed  | Accuracy | RAM Required | Best For |
|--------|--------|----------|--------------|----------|
| tiny   | Fast   | Low      | ~1 GB        | Quick drafts, testing |
| base   | Medium | Good     | ~1 GB        | **Default** - good balance |
| small  | Slow   | Better   | ~2 GB        | Important meetings |
| medium | Slower | High     | ~5 GB        | Critical recordings |
| large  | Slowest| Highest  | ~10 GB       | Professional transcription |

## Output Format

The generated markdown includes:

```markdown
# Meeting Notes

**Date:** 2026-02-03 14:30
**Source:** meeting.wav

## Attendees
- John
- Sarah

## Key Points
- Discussed Q1 roadmap priorities
- Agreed to focus on mobile app performance
- Budget approved for new infrastructure

## Action Items
- [ ] John to draft the mobile optimization proposal
- [ ] Sarah to schedule follow-up meeting with stakeholders
- [ ] Review infrastructure costs by Friday

## Full Transcript
[Complete transcription text...]

---
*Generated by voice-notes on 2026-02-03 14:30:45*
```

## Python API

You can also use voice-notes as a library:

```python
from voice_notes.transcriber import Transcriber
from voice_notes.note_formatter import NoteFormatter

# Transcribe audio
transcriber = Transcriber(model_name="base")
result = transcriber.transcribe("meeting.wav")

# Format notes
formatter = NoteFormatter(audio_filename="meeting.wav")
key_points = formatter.extract_key_points(result["text"])
action_items = formatter.extract_action_items(result["text"])
attendees = formatter.extract_attendees(result["text"])

# Generate markdown
markdown = formatter.format_markdown(
    transcription=result["text"],
    key_points=key_points,
    action_items=action_items,
    attendees=attendees
)

print(markdown)
```

## Integration with voice-mcp

When using `--speak`, voice-notes uses the voice-mcp TTS system at:
`/mnt/d/_CLAUDE-TOOLS/voice-mcp/speak.py`

This provides a spoken summary including:
- Word count of transcription
- Number of key points found
- Number of action items identified
- The main key point
- The first action item

## Troubleshooting

### "No module named 'whisper'"
```bash
pip install openai-whisper
```

### "FFmpeg not found"
Install FFmpeg for your operating system (see Prerequisites above).

### Out of Memory
Use a smaller model:
```bash
voice-notes file.wav --model tiny
```

### Slow Transcription
- Use `--model tiny` or `--model base` for faster processing
- Ensure you have a GPU with CUDA support for faster inference
- For CPU-only systems, expect ~1x realtime for base model
