# Prime Bug - Bug Investigation Focus

## Purpose
Set up context for efficient bug diagnosis and fixing

## Workflow
1. Read error logs if provided
2. Identify affected components
3. Trace data flow through the system
4. Check related tests
5. Review recent changes in affected areas

## Variables
- BUG_DESCRIPTION: Description of the bug
- ERROR_MESSAGE: Specific error message if available
- AFFECTED_AREA: Component/module affected

## Report
- Root cause analysis
- Affected systems
- Proposed fix approach
- Test coverage gaps
