# Prime Feature - Feature Development Focus

## Purpose
Prepare context for implementing new features

## Workflow
1. Read existing feature patterns in src/features/
2. Identify component structure and conventions
3. Check API/service integration patterns
4. Review state management approach
5. Examine test patterns for features

## Variables
- FEATURE_NAME: Name of the feature to implement
- FEATURE_AREA: Area of codebase (e.g., UI, API, services)

## Report
Summarize:
- Relevant existing patterns
- Dependencies needed
- Test requirements
- Integration points
